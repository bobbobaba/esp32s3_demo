# 更新记录

本文件记录公开仓库中的主要更新内容。每次功能变更、UI 调整、硬件支持或文档更新都应同步追加简要说明。

## 2026-07-24

### 信号强度改为像素WiFi扇形图标

- 参考用户提供的像素 WiFi 图标样式，手工逐像素设计 22x20 图标：三条信号弧 + 金边宝箱底座（宝箱开箱放光的意象）。
- 4 根信号柱替换为 5 状态 WiFi 图标：断网时弧线暗、宝箱锁扣灰暗；信号 1 格点亮宝箱金扣，2-4 格再从内到外点亮白青色弧线。

### 首页时间改为胖像素数字

- 参考用户提供的像素表盘样式，程序化绘制 24x32 胖像素数字精灵（0-9）和 12x32 冒号：深梅色字身、左上粉色高光、浅蓝色描边。
- 首页七段数码管时间替换为数字图片，`setHomeTimeDigit` 改为按字符切换图片源，无效字符隐藏对应数字位。
- 删除不再使用的七段管绘制代码和 `home_time_segments`、`home_time_colon_dots` 等对象字段。

### 首页血条蓝条与冒险岛图标按钮

- CPU/MEM 文字百分比改为冒险岛式状态条：CPU 后跟红色血条、MEM 后跟蓝色魔法条，按百分比伸缩，数据无效时隐藏填充。
- 天气图标从 16x16 AI 生成图换为程序化绘制的 24x24 简洁像素图标（太阳/多云/雨/雪），块状描边更清晰。
- 删除 P4 MENU / P7 AI 文字提示，改为 22x22 冒险岛风格棕色像素按钮（菜单三横线、AI 对话气泡）。
- 全部描边和文字阴影统一为深棕色，温度数值改为金色，信号柱改为绿色/灰蓝配色，更贴近冒险岛 UI 配色。

### 首页去掉白色卡片，改为描边像素文字

- 删除首页 5 块半透明白色卡片，像素背景完整可见，整体更接近冒险岛复古风格。
- 首页文字改为白字加深色阴影（描边效果），七段数码管时间和冒号改为白色填充加深色描边，信号柱同样加描边并改为白色/灰蓝配色。
- 修复 `lvglSetLabel` 在动态刷新时把标签字体重置回 Montserrat 默认字体的问题：unscii/simsun 点阵标签保持点阵字体，并同步刷新阴影副本文本。

### 界面文字改用像素点阵字体

- 启用 LVGL 内置 unscii 8x8 复古点阵字体，所有 ASCII 界面文字（状态、菜单、按键提示等）由 Montserrat 换成像素风格，中文继续使用 simsun 16 CJK。
- unscii 为 8px 等宽字体，统一设置 -1 字距压缩到 7px/字符，兼容原有标签宽度；加宽首页 CPU/MEM 卡片，微调设置页与 LED 页底部按键提示布局，避免文字被裁剪。

### 首页改为冒险岛像素风表盘

- 新增 `src/ui/pixel_watch_img.c/.h`：128x128 白天/夜晚像素背景、48x48 三帧小猫动画、16x16 天气图标（晴/多云/雨/雪），RGB565 格式约 87KB。
- 首页背景改为 8 位像素风场景（蘑菇屋、小桥、草地），文字和七段数码管时间放在半透明白色卡片上保证可读性。
- 天气显示由多个 shape 拼图改为单个 16x16 像素图标，按天气代码切换晴/多云/雨/雪，新增降雪判断（代码 71-77、85-86）。
- 新增穿红色连帽衫的白色小猫待机动画（甩尾/眨眼，600ms 一帧），仅首页可见时刷新。
- 按本地时间 7:00-19:00 显示白天背景，其余时间切换为星空夜景（小猫睡觉、蘑菇屋亮灯），夜间隐藏站立小猫。
- EEZ 编辑项目（eez-ui）暂未同步本次像素背景改动。

### 首页改为白底无框大时间

- 删除首页 CPU/MEM、时间、温度、P4/P7 周围的彩色背景框，让文字直接显示在白色背景上。
- 启用 48px 时间字体，首页时间放大显示。
- 首页文字颜色改为适配白色背景的深色高对比配色，小字提升到 12px，Wi-Fi 信号柱改为深色/浅灰状态。
- 首页天气区域去掉 `TEMP` 文本，改为天气图形符号加温度数字；运行时按天气代码切换太阳、云、雨。
- 首页时间拆成小时、冒号、分钟 3 个对象，小时和分钟强制使用 48px 字体，避免整串时间在小屏上被裁剪后看起来没有放大。
- 首页时间改为七段数码管图形，不再依赖字体渲染，修复字体大小不变和错位问题。

### 首页信号改为柱状图

- 右上角 Wi-Fi 状态从文字改为 4 格信号强度柱状图。
- 固件运行时根据 `WiFi.RSSI()` 动态点亮信号格，未联网时全部置灰。
- EEZ Home 预览同步使用 4 个小 panel 模拟信号柱。

### 调整首页信息布局

- 首页左上角改为 CPU/MEM 两行状态，右上角显示 Wi-Fi 信号状态。
- 时间移动到中间偏左并保持大字号，温度缩小后放到时间下方，P4/P7 提示继续向下排列。
- 同步更新 EEZ Home 预览布局，保持小猫在底层、文字和卡片在上层。

### 修复小猫遮挡首页文字

- 首页继续保持白色背景，但小猫图片恢复透明背景，避免白色图片区遮挡卡片和文字。
- 调整 LVGL 和 EEZ Home 页面层级，小猫作为底层装饰，状态卡片和文字显示在上层。

### 统一首页白色背景

- 将 LVGL 页面基础背景改为白色覆盖，避免手表实际显示和 EEZ 预览背景不一致。
- 将首页小猫资源改为不透明白底 80x80 图片，消除透明区域在 EEZ 中显示点阵/棋盘背景的问题。

### 放大并镜像首页小猫

- 将首页小猫资源从 40x40 改为 80x80，并做左右镜像处理。
- 同步调整固件首页和 EEZ 项目中的图片控件位置，小猫固定在右下角显示。

### 取消固件整屏深色背景

- 移除 LVGL 页面基础层强制 `0x242832` 深灰铺底，改为透明背景，保持手表实际显示和 EEZ 无背景预览一致。
- EEZ 项目继续保留无背景 screen 设置，首页小猫和卡片控件单独显示。

### 重做首页并加入小猫

- 使用本地小猫图片生成 40x40 透明 LVGL 图片资源，放到首页右下角。
- 重排首页为左侧时间、温度、CPU/MEM 状态，右侧保留小猫和按键提示，减少文字互相遮挡。
- 同步更新 EEZ UI 镜像目录，保持固件实际页面和 EEZ 导出目录一致。
- 将小猫图片作为 `catstand` bitmap 嵌入 `.eez-project`，EEZ Studio 首页编辑器中也能看到图片控件。

### 恢复稳定中文字体

- 移除 12px 自定义中文字库，AI 回复仍优先使用服务端返回的中文位图，避免引入全量中文字库。

### 修复 EEZ 中文菜单显示

- 固件动态中文 label 和 EEZ 静态中文 label 统一使用 `lv_font_cn_12`，避免中文被裁剪为空白。

### 同步 EEZ 灰色背景到固件

- 修正 EEZ 生成页面在固件中仍使用近黑色 `0x05070B` 背景的问题。
- 将固件实际 UI 目录和 EEZ 导出镜像目录的页面底色统一为中性灰 `0x242832`。
- 重新接回 EEZ 菜单页，菜单 label 支持精简中文字体，EEZ 预览和烧录后的菜单文字保持一致。
- 首页底部左右数据格改为统一灰色数据块，避免看起来像倒置的信号格。

### 修复菜单中文空白

- 菜单中文继续使用固件内置 16x16 点阵字库，避免 EEZ 动态 label 切换精简 LVGL 字库后出现空白。

### 修复中文方框缺字

- 关闭内置宋体和大字体索引，AI 通话页继续使用服务端返回的中文回复位图，降低 Flash 占用。

### 增加中文字体和私有服务器构建方式

- 本地烧录使用 PlatformIO 临时编译参数接入私有服务器，公开仓库继续保留脱敏默认地址和示例配置。

- 新增 `UiPage` 到 EEZ `ScreensEnum` 的映射：首页、菜单、AI 通话、设置、服务器、LED、Wi-Fi 配网和 MPU 数据页优先使用 EEZ 页面。

- MPU 水平仪、姿态角、里程计和事件页暂时保留原手写 UI，作为复杂传感器页面的稳定兜底。

- 当前 EEZ UI 仍是设计草图，主固件暂时继续使用现有手写 LVGL 页面。

### 实现服务器 OTA 流程

- 新增 `serviceOta()`，联网空闲时定时检查服务器设备配置。
- 新增 `checkAndApplyOta()`，读取 `device-config` 中的固件版本和下载地址。
- 新增 `applyOtaFirmware()`，使用 `Update.writeStream()` 下载并写入新固件。
- 设置页显示 OTA 状态，例如 `OTA OFF`、`OTA OK`、`OTA WRITE`、`OTA REBOOT`。
- 公开版默认 `example.invalid` 会跳过 OTA，不包含真实服务器地址。

### 批量拆分页面动态刷新和完善开源文档

- 设置页拆分为框架和动态刷新，音量变化只刷新音量数字和进度条。
- 服务器页拆分为框架和动态刷新，运行时间、负载、内存、磁盘只刷新数据区域。
- LED 页拆分为框架和动态刷新，灯效切换和动画颜色只刷新预览区和选中状态。
- Wi-Fi 二维码页复用 LVGL 对象，避免重复清屏重建二维码页面。
- 增加音量持久化，按键调整后的音量会保存到 ESP32 Preferences。
- 新增硬件接线、EEZ Studio 接入计划和 OTA API 设计文档。

### 拆分菜单页动态刷新

- 将菜单页拆分为 `renderMenuFrame()` 和 `renderMenuList()`。
- 新增菜单渲染缓存，记录上次选中项和列表窗口起点。
- 菜单上下选择时，如果列表窗口没有滚动，只重画原选中行和新选中行。
- 为后续 EEZ Studio 菜单页面接入保留更清晰的列表刷新入口。

### 拆分 AI 通话页动态渲染

- 将 AI 通话页拆分为页面框架和动态内容区。
- 新增 `VoiceRenderMode`，区分上传、录音、播放、结果和待机状态。
- 录音计时只刷新中间内容区，减少整屏清空造成的闪烁。
- 切换页面时重置语音页渲染缓存，避免残影或状态误判。

### 增加统一按键调度

- 将 `serviceButtons()` 收口为按键扫描和消抖逻辑。
- 新增 `dispatchButton(index)` 统一入口。
- 按页面拆分按键处理函数，便于后续接入 EEZ Studio/EEZ Flow 页面回调。

### 重构 UI 页面状态

- 用单一 `UiPage currentUiPage` 替换多个 `showXxxPage` 布尔变量。
- 新增 `setUiPage()` 和 `sensorPageVisible()`。
- 页面跳转状态更集中，便于后续迁移到 LVGL/EEZ 生成页面。
